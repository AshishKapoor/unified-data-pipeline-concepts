Ch 01 — The Unified Model: Why Beam Exists

One pipeline definition, many runners. Batch is a bounded special case of streaming — Beam = Batch + strEAM.

The problem Beam dissolves

Before Beam, teams maintained two codebases for the same logic: a batch job (for historical, bounded data) and a streaming job (for live, unbounded data) — the classic Lambda architecture. The two used different APIs, different engines, and drifted apart over time.

Apache Beam's insight is that batch is just streaming over a bounded dataset. Express your computation once against a single model, and a pluggable runner translates it to a concrete engine: the local DirectRunner, the FlinkRunner, Google Cloud Dataflow, Spark, and more.

🎯
The one idea for this chapter: the transform graph is independent of the engine. In this course we run on Flink, but the exact same Python would run on Dataflow by changing a single flag — --runner.

Beam APIs introduced

  • apache_beam.Pipeline — the container you attach transforms to.
  • beam.io.ReadFromText — a bounded source reading lines from files.
  • beam.FlatMap — 1 input → N outputs (here: a line → its words).
  • combiners.Count.PerKey — a built-in aggregation.
  • --runner — the single flag that selects the execution engine.

The example pipeline

Canonical WordCount. Read → tokenize → count → format → write — engine-agnostic:

with beam.Pipeline(options=options) as p:
    counts = (
        p
        | "ReadLines"    >> beam.io.ReadFromText(known.input)
        | "Tokenize"     >> beam.FlatMap(split_words)      # 1 line -> N words
        | "PairWithOne"  >> beam.Map(lambda w: (w, 1))
        | "CountPerWord" >> combiners.Count.PerKey()        # the aggregation
        | "Format"       >> beam.MapTuple(lambda w, n: f"{w}: {n}")
    )
    counts | "WriteCounts" >> beam.io.WriteToText(known.output, file_name_suffix=".txt")

The submitter ships this graph to the Beam Flink Job Server, which turns it into a Flink JobGraph and submits it to the JobManager. Open the Flink UI to watch the job execute.
