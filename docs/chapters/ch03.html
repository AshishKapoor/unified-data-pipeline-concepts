Ch 03 — Element-wise Transforms & the DoFn Lifecycle

Map, FlatMap and Filter are sugar over one primitive — ParDo + a DoFn. The runner drives every DoFn through a fixed lifecycle, and a bundle is its unit of commit and retry.

Concept

Almost every element-wise transform you write reduces to a single primitive: beam.ParDo applied to a DoFn. The friendlier transforms are just thin wrappers:

  • beam.Map(fn) — a 1 → 1 ParDo (each input yields exactly one output).
  • beam.FlatMap(fn) — a 1 → N ParDo (each input yields an iterable, flattened).
  • beam.Filter(pred) — a keep-or-drop ParDo (yields the element, or nothing).

Reach for ParDo(DoFn) directly when you need more than a pure function: setup of an expensive client, per-batch buffering, multiple outputs, access to the element's timestamp or window, or state & timers (Ch 13). The DoFn is the general case; the rest are conveniences.

The lifecycle the runner guarantees

The runner does not just call process. It drives each DoFn instance through an ordered set of callbacks. Internalising this ordering is what separates correct element-wise code from subtly broken code: 

setup()                 once per instance     → open reusable resources (clients, sessions)
  start_bundle()        once per bundle        → init per-bundle state / buffers
    process(elem)       once per element       → the work; yields 0..N outputs
    process(elem)       ...
  finish_bundle()       once per bundle        → flush the batch (commit-tied)
teardown()              once per instance      → close resources (BEST-EFFORT, may not run)
🎯
The one idea for this chapter: a bundle is the runner's unit of commit and retry. The runner — not your code — decides how the element stream is sliced into bundles. On Flink, bundle boundaries track checkpoints and network buffers; they have nothing to do with the shape of your Create list.
⚠️
Idempotency is non-negotiable. If any element in a bundle fails, the entire bundle is replayed from start_bundle. So process must be safe to re-run: prefer keyed/upsert writes over blind inserts, and never mutate global counters that would double-count on retry.
🧹
teardown is best-effort. A hard worker crash can skip it entirely. Use it only for resource hygiene (closing a client). Anything that must not be lost — flushing a batch of writes — belongs in finish_bundle, which runs on the bundle's commit path.

Beam APIs introduced

  • beam.ParDo — the element-wise primitive that drives a DoFn.
  • beam.DoFn — subclass and override the lifecycle methods.
  • setup / start_bundle / process / finish_bundle / teardown — the lifecycle callbacks.
  • DoFn.TimestampParam — ask the runner to inject each element's event timestamp into process.
  • beam.Filter / beam.Map — the sugar transforms, shown alongside the raw ParDo.

The example pipeline

A LifecycleDoFn logs every callback as it fires, so the worker logs reveal bundle boundaries directly. It opens a fake client in setup, buffers per bundle, and flushes in finish_bundle. We feed it ~20 Create'd records: 

class LifecycleDoFn(beam.DoFn):
    def setup(self):                      # once per instance — open reusable resources
        self._client = FakeClient(worker_tag=f"pid{os.getpid()}")
        self._bundle_no = 0

    def start_bundle(self):               # once per bundle — reset per-bundle buffer
        self._bundle_no += 1
        self._buffer = []

    def process(self, element, timestamp=beam.DoFn.TimestampParam):
        enriched = self._client.enrich(element)
        self._buffer.append(enriched)     # batch now…
        yield enriched                    # 1→1 here, but a DoFn may yield 0..N

    def finish_bundle(self):              # once per bundle — flush (commit-tied)
        log.info("flush bundle #%d: %d records", self._bundle_no, len(self._buffer))

    def teardown(self):                   # best-effort — close, never flush
        self._client.close()

with beam.Pipeline(options=options) as p:
    (p
     | "CreateRecords" >> beam.Create(records)               # ~20 bounded records
     | "DropDates"     >> beam.Filter(lambda r: r["fruit"] != "date")   # Filter = sugar
     | "Lifecycle"     >> beam.ParDo(LifecycleDoFn())        # the general primitive
     | "Format"        >> beam.Map(lambda r: f"id={r['id']} …"))        # Map = sugar
