Ch 12 — Late Data, Allowed Lateness & Dropped Data

A window does not close the instant the watermark passes its end. allowed_lateness keeps its state alive for a grace period so stragglers still update the result — until the watermark crosses window_end + allowed_lateness, when the state is garbage-collected and any later element is dropped silently. The fix for "silent" is a metric.

The late-element lifecycle

In Ch 10 we learned the watermark is the runner's estimate of "event time has progressed at least this far — I don't expect anything older." When the watermark passes a window's end, Beam fires the ON_TIME pane. But "don't expect" is not "guarantee": a slow network path or a re-delivered message can carry an event whose timestamp falls inside a window the watermark has already swept past. That element is late.

What happens to a late element depends entirely on one window setting:

  • Within the grace period — if the watermark has passed window_end but not yet window_end + allowed_lateness, the window's persisted state still exists. The late element folds in and the window fires again as a LATE pane, updating the result.
  • Beyond the grace period — once the watermark crosses window_end + allowed_lateness, the runner garbage-collects the window's state to bound memory. There is nothing left to update, so any element that arrives even later is dropped. It never reaches your aggregation and emits no output.
🎯
The one idea for this chapter: allowed_lateness is the price you pay (in retained state) to tolerate stragglers. Bigger grace = fewer drops but more memory and longer-lived windows. It is a correctness ↔ resource trade-off you set per window.
⚠️
Drops are silent by default. A dropped element produces no error, no log, no output — it just vanishes. If you don't measure it you'll never know your results are quietly incomplete. Always attach a counter (below) and alert when it moves.

Reading the pane: EARLY / ON_TIME / LATE

Every firing carries a PaneInfo describing why it fired. Inside a DoFn you read it with DoFn.PaneInfoParam. The fields that matter here:

  • pane.timing — EARLY (fired before the watermark, e.g. a speculative trigger), ON_TIME (the watermark just passed the window end), or LATE (a late element within lateness re-fired the window).
  • pane.index — 0 for the window's first firing, then 1, 2, … for each subsequent firing. A LATE pane always has a higher index than the ON_TIME pane.
  • pane.is_first / pane.is_last — whether this is the first / final pane the window will ever emit.

Beam APIs introduced

  • allowed_lateness= on beam.WindowInto — the grace period (a Duration) during which late elements still update the window.
  • AfterWatermark(late=…) — fire ON_TIME at the watermark, then once more for each late element that arrives within lateness.
  • DoFn.PaneInfoParam — injects the firing's PaneInfo (timing + index) into process().
  • beam.metrics.Metrics.counter(namespace, name) — a worker-aggregated counter; here it makes silent drops observable.
  • AccumulationMode.ACCUMULATING — each LATE pane re-emits the full running total so you can see the late element actually updated the result.

The example pipeline

One 1-minute FixedWindow with a 30s grace. Three on-time events form the ON_TIME pane; a straggler at +10s re-fires it (LATE pane, sum grows); a straggler at +90s lands past the GC horizon and is counted as dropped: 

windowed = (
    p
    | "Replay"         >> ReplayEvents(records, emission_order=emission_order)
    | "TagLateDropped" >> beam.ParDo(TagLateAndDropped())   # counts late / dropped via Metrics
    | "ToKV"           >> beam.Map(lambda r: (r["key"], r["amount"]))
    | "Window" >> beam.WindowInto(
          FixedWindows(60),                                  # tumbling 1-minute windows
          trigger=AfterWatermark(late=AfterWatermark()),     # ON_TIME, then re-fire on each late elem
          allowed_lateness=Duration(seconds=30),             # 30s grace before state is GC'd
          accumulation_mode=AccumulationMode.ACCUMULATING,   # each pane re-emits the running sum
      )
    | "SumPerKey"  >> beam.CombinePerKey(sum)
    | "FormatPane" >> beam.ParDo(FormatPane())               # reads DoFn.PaneInfoParam (timing+index)
)

The FormatPane DoFn logs pane.timing and pane.index for each firing, and TagLateAndDropped increments the late_elements / dropped_elements counters that surface in the run log and the Flink UI.
