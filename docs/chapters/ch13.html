Ch 13 — Stateful Processing: State & Timers in DoFn

Give a DoFn private, durable memory — state cells scoped per key, per window — plus timers it can arm to wake itself up in event time or processing time. These are the raw primitives every streaming feature is built on.

Concept: from stateless to stateful

Everything so far was effectively stateless from your code's point of view: GroupByKey, Combine, and windowing carry state for you, but your functions never touched a cell directly. A stateful DoFn opens the lid. For each (key, window) the runner hands your DoFn a small set of private state cells and lets it schedule timers. The runner shuffles by key first, so each key's state lives on exactly one worker — user-A's buffer never sees user-B's events.

This is how you implement sessionization, deduplication, join-with-timeout, rate limiting, and fraud heuristics by hand, with full control over what you remember and when you act on it.

The three state cells we use

  • ReadModifyWriteStateSpec → a ValueState: one mutable slot you can read() / write(v) / clear(). Here it remembers the latest event time seen for the user.
  • BagStateSpec → a BagState: an append-only multiset you add() to, then read() + clear(). Here it buffers the raw events of the in-progress session.
  • CombiningValueStateSpec → a CombiningState: a running aggregate folded through a CombineFn (we use sum). The runner keeps only the accumulator — far cheaper than counting the whole bag.
🎯
The one idea for this chapter: state and timers are automatically keyed and windowed by the runner. You never look up a key — you just read and write "your" cells, and the runtime guarantees they belong to the element's key and window.

Timers: waking yourself up

A TimerSpec declares a timer in a time domain. A method decorated with @on_timer(SPEC) is the callback the runner invokes when it fires — and inside that callback you may read/clear the same cells and yield output.

  • TimeDomain.WATERMARK — fires when the watermark passes the set event-time instant. Perfect for "the session has been idle for N seconds of event time". This pipeline uses exactly this.
  • TimeDomain.REAL_TIME — fires at a wall-clock (processing-time) instant, regardless of where the watermark is.

Re-setting a timer to a new instant simply overwrites the old one, so a steady stream keeps pushing an idle-gap deadline out — the timer only fires after a genuine lull.

Beam state → Flink checkpointed state

On the portable Flink runner these cells become Flink keyed, checkpointed state (ValueState / ListState / AggregatingState) in the configured state backend, and timers become Flink event-time / processing-time timers. So everything here participates in checkpointing and is restored exactly-once after failure (Ch 16).

⚠️
Stay on the mature trio. The exotic Beam state types (OrderedListState, MultimapState, SetState) are still less mature in Python-on-Flink. This course sticks to the rock-solid Value / Bag / Combining cells, which map cleanly onto Flink state. Also: always clear() your cells when a session ends — otherwise per-key state grows without bound, the classic stateful-pipeline memory leak.

Beam APIs introduced

  • apache_beam.transforms.userstate — home of the state & timer specs.
  • ReadModifyWriteStateSpec / BagStateSpec / CombiningValueStateSpec — the three cell types.
  • TimerSpec(name, TimeDomain.WATERMARK) — declare an event-time timer.
  • beam.DoFn.StateParam / beam.DoFn.TimerParam — inject the cells/timers into process.
  • @on_timer(SPEC) — the callback the runner fires when a timer matures.

The example pipeline

A per-user "session / fraud-ish" detector. Each event buffers into a BagState, bumps a CombiningState count, and re-arms an event-time gap timer. When the watermark proves the user has been idle for GAP seconds, @on_timer fires, drains the bag, emits one session summary, and clears every cell: 

class SessionizingDoFn(beam.DoFn):
    BUFFER  = BagStateSpec("buffer", beam.coders.PickleCoder())     # raw events
    COUNT   = CombiningValueStateSpec("count", VarIntCoder(), sum)  # running count
    LAST_TS = ReadModifyWriteStateSpec("last_ts", FloatCoder())     # newest event time
    GAP_TIMER = TimerSpec("gap_timer", TimeDomain.WATERMARK)        # event-time idle timer

    def process(self, element, timestamp=beam.DoFn.TimestampParam,
                buffer=beam.DoFn.StateParam(BUFFER),
                count=beam.DoFn.StateParam(COUNT),
                last_ts=beam.DoFn.StateParam(LAST_TS),
                gap_timer=beam.DoFn.TimerParam(GAP_TIMER)):
        _user, event = element
        buffer.add(event)                 # BagState.add
        count.add(1)                      # CombiningState folds via sum
        last_ts.write(float(timestamp))   # ValueState.write
        gap_timer.set(timestamp + Duration(seconds=SESSION_GAP_SEC))  # (re)arm the idle gap

    @on_timer(GAP_TIMER)
    def on_gap(self, buffer=beam.DoFn.StateParam(BUFFER),
               count=beam.DoFn.StateParam(COUNT),
               last_ts=beam.DoFn.StateParam(LAST_TS)):
        events, total = list(buffer.read()), count.read() or 0
        flag = "BURST" if total >= BURST_THRESHOLD else "ok"
        buffer.clear(); count.clear(); last_ts.clear()   # always clear, or state leaks
        yield (flag, {"events": total, "flag": flag})

The input must be keyed — stateful DoFns require a KV so the runner can shuffle by key. We key the synthetic stream by user, drop it in the global window (sessions are delimited by our timer, not by windowing), then ParDo the stateful DoFn.
