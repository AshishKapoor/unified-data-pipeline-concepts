Ch 06 — Efficient Aggregation: Combine, CombineFn, Built-ins

Combine beats GroupByKey + reduce because an associative/commutative CombineFn lets the runner pre-aggregate before the shuffle — moving tiny accumulators instead of every raw element.

Why not just GroupByKey?

In Chapter 5 we grouped a keyed PCollection with GroupByKey and reduced the resulting iterable. Correct — but wasteful. GroupByKey must ship every element to the worker that owns its key before any reduction can happen. If a sensor emits a million readings, a million readings cross the network just so you can average them.

Combine exploits a structural fact: if your reduction is associative ( (a∘b)∘c = a∘(b∘c) ) and commutative ( a∘b = b∘a ), it can be applied in any order and in any grouping — including partially, on each worker, before the shuffle. The runner reduces each bundle down to a small accumulator and ships only that. This is combiner lifting (a.k.a. partial / pre-aggregation), and Flink inserts the local pre-combine for you automatically.

🎯
The one idea for this chapter: a CombineFn contracts that your aggregation is associative + commutative. That single promise lets the runner split the work into a cheap pre-shuffle partial combine and a post-shuffle final combine — cutting data movement, often dramatically.

The 4-method CombineFn

Every beam.CombineFn is four methods around an accumulator — an intermediate value whose size stays bounded no matter how many inputs arrive. For a mean, the accumulator is a (sum, count) pair:

  • create_accumulator() — return a fresh, empty accumulator (the identity element). For mean: (0.0, 0).
  • add_input(acc, element) — fold one input into an accumulator. Runs per-element, per-bundle, before the shuffle.
  • merge_accumulators(accs) — combine several partial accumulators into one. Runs both pre-shuffle (per bundle) and post-shuffle (per key).
  • extract_output(acc) — turn the final accumulator into the answer. For mean: sum / count.
⚠️
You cannot average averages. Carrying a bare running mean is not associative. The fix is the accumulator shape: carry (sum, count) — sums and counts add freely in any order — and only divide in extract_output. Choosing the right accumulator is the whole craft of writing a CombineFn.

Global vs per-key, and the built-ins

The same CombineFn serves two shapes. beam.CombineGlobally(fn) reduces an entire PCollection to a single value; beam.CombinePerKey(fn) reduces per key on a PCollection of (key, value) pairs. Beam ships a battery of ready-made combiners in beam.combiners so you rarely write the boilerplate:

  • combiners.Mean.PerKey() / Mean.Globally() — exactly the (sum, count) CombineFn we write by hand below.
  • combiners.Count.PerKey() / Count.Globally() — how many elements (the WordCount aggregation from Ch 1).
  • combiners.Top.Of(n) / Top.PerKey(n) — the n largest, kept in a bounded heap accumulator.
  • combiners.Sample.FixedSizeGlobally(n) — a uniform reservoir sample of n elements.
  • beam.Distinct() — deduplicate (a combine-flavoured dedup over the shuffle).

Beam APIs introduced

  • beam.CombineFn — the 4-method base class: create_accumulator, add_input, merge_accumulators, extract_output.
  • beam.CombinePerKey — per-key aggregation with combiner lifting.
  • beam.combiners.Mean / Count / Top — built-in CombineFns.
  • beam.Values — drop keys to feed a global combine.
  • beam.Flatten — union the two labelled result streams to compare them.

The example pipeline

A hand-written AverageFn computes the per-sensor mean; the built-in Mean.PerKey() computes it again on the same input to prove they agree: 

class AverageFn(beam.CombineFn):
    def create_accumulator(self):            # (1) identity: (sum, count)
        return (0.0, 0)

    def add_input(self, acc, element):       # (2) fold one reading — runs pre-shuffle
        s, c = acc
        return (s + element, c + 1)

    def merge_accumulators(self, accs):      # (3) combine partials — pre AND post shuffle
        sums, counts = zip(*accs)
        return (sum(sums), sum(counts))

    def extract_output(self, acc):           # (4) final answer
        s, c = acc
        return s / c if c else float("nan")

with beam.Pipeline(options=options) as p:
    readings = p | "SensorReadings" >> beam.Create(SAMPLE_READINGS)   # (sensor_id, reading)

    custom  = readings | "AverageByHand" >> beam.CombinePerKey(AverageFn())
    builtin = readings | "AverageBuiltin" >> combiners.Mean.PerKey()   # same (sum,count) inside
    # ... Flatten + format both paths to show identical means ...
