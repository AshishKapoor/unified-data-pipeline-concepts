Ch 02 — Core Abstractions: Pipeline, PCollection, PTransform, PValue

Beam has only four load-bearing nouns. Master them and every transform you ever write reads cleanly — and every element secretly carries value + event-timestamp + window(s) + pane-info.

Concept — the four primitives

Apache Beam's API surface is large, but its conceptual core is tiny. Four types do all the work, and they fit together like nouns in a sentence:

  • Pipeline — the container and the directed acyclic graph you attach transforms to. Crucially, building a pipeline does nothing: it merely records the graph. Execution is deferred until the with-block exits and the runner takes over.
  • PValue — the abstract thing that flows between nodes. Its start-of-graph form is PBegin (what you get from p | …); its everyday form is a…
  • PCollection — an immutable, distributed, possibly-unbounded bag of elements. It is emphatically not a Python list: you cannot index it, its order is undefined, and it carries two pieces of hidden state — a coder (how to serialize elements between workers) and a windowing strategy.
  • PTransform — a graph node that consumes PValues and produces PValues. You apply one with the pcoll | "Label" >> transform operator.
🎯
The one idea for this chapter: a PCollection is not a collection of bare values. Every element is conceptually a 4-tuple (value, event-timestamp, window(s), pane-info) — and those fields exist even in a plain bounded job where you never set them.

Composite transforms — abstraction in Beam

Reuse in Beam comes from composite PTransforms. You subclass beam.PTransform and override expand(), which receives the input PCollection and must return a PValue. Inside, you wire together smaller inner transforms. Beam's own built-ins (combiners.Count, GroupByKey wrappers, IO connectors) are all composites — it is composites all the way down.

The labels you attach with >> are not cosmetic: they nest. An inner "AddScore" applied inside a composite labelled "ScoreAndKeep" renders in the Flink/Dataflow UI as a collapsible sub-graph, so a 200-node pipeline reads like an outline.

⚠️
Immutability is a contract, not a guard. Beam will not stop you mutating a dict in place inside a Map, but doing so is undefined behaviour once the element is shared, cached for a retry, or fused across stages. Always build and return a new object.

The anatomy of an element

Because we never set a timestamp or window, it is tempting to think the metadata is absent. It isn't. We expose it with a DoFn that asks the runner to inject the implicit fields via the special default parameters beam.DoFn.TimestampParam, beam.DoFn.WindowParam, and beam.DoFn.PaneInfoParam. For records created by beam.Create with no explicit timestamp you will see the timestamp pin to MIN_TIMESTAMP and the window resolve to the single GlobalWindow — the defaults every Beam element is born with. Beam APIs introduced

  • beam.Create — turn an in-memory iterable into a bounded PCollection.
  • beam.PTransform + expand() — build a custom composite node.
  • beam.Map / beam.Filter — the two inner transforms (1→1, 1→0|1).
  • beam.ParDo + beam.DoFn — the general element-wise transform.
  • beam.DoFn.TimestampParam, WindowParam, PaneInfoParam — inject an element's implicit metadata into process().

The example pipeline

A custom composite (Map then Filter) hidden behind one labelled node, followed by a pass-through DoFn that logs each element's four layers: 

class ScoreAndKeep(beam.PTransform):
    """A composite: enrich (Map) then drop low scorers (Filter)."""
    def expand(self, pcoll):
        return (
            pcoll
            | "AddScore"      >> beam.Map(self._add_score)      # 1 -> 1
            | "DropLowScores" >> beam.Filter(                   # 1 -> 0 or 1
                lambda rec: rec["score"] >= self._threshold)
        )

class LogElementAnatomy(beam.DoFn):
    def process(self, element,
                timestamp=beam.DoFn.TimestampParam,   # event-timestamp
                window=beam.DoFn.WindowParam,         # window(s)
                pane=beam.DoFn.PaneInfoParam):        # pane-info
        logging.info("value=%r ts=%s window=%s pane=%s",
                     element, timestamp, window, pane)
        yield element                                  # pass through unchanged

with beam.Pipeline(options=options) as p:              # deferred: nothing runs yet
    created = p       | "Create"       >> beam.Create(records)
    kept    = created | "ScoreAndKeep" >> ScoreAndKeep(known.threshold)
    _       = kept    | "LogAnatomy"   >> beam.ParDo(LogElementAnatomy())

The submitter ships this graph to the Beam Flink Job Server, which expands the composite into its inner Map/Filter and submits the resulting JobGraph to the JobManager. Open the Flink UI to watch the job — the composite appears as a single collapsible operator.
